# How to Operate the XAI Financial Services Project

Complete steps to run the whole project: **training** → **backend** → **frontend**.

---

## One-time setup (do this first)

1. **Install everything** (see [INSTALL.md](INSTALL.md)):
   - Create venv: `python -m venv venv`
   - Activate: `.\venv\Scripts\Activate.ps1`
   - Install Python: `pip install -r training\requirements.txt` and `pip install -r backend\requirements.txt`
   - Install frontend: `cd frontend` then `npm install`

2. **Put your datasets** in one folder (e.g. `C:\Users\97254\Desktop\datasets`):
   - `loan_data_set.csv`
   - `american_bankruptcy.csv`
   - From **GiveMeSomeCredit.zip**: extract and add `cs-training.csv` (or the main CSV)

3. **Run training once** (so the backend has models to load):
   - See **Step 1** below.

---

## Every time you want to use the app

Do **Step 1** only if you changed data or want to retrain. Otherwise do **Step 2** then **Step 3**.

---

### Step 1 — Training (optional each time)

**When:** Once after setup, or whenever you change data / want to retrain.

**Terminal:** PowerShell, project root.

```powershell
cd C:\Users\97254\Desktop\Finance_XAI
.\venv\Scripts\Activate.ps1
$env:XAI_DATA_DIR = "C:\Users\97254\Desktop\datasets"
cd training
python preprocess_loan.py
python preprocess_bankruptcy.py
python preprocess_credit_risk.py
python train_models.py
cd ..
```

**Result:** Files in `training\output\` (loan, bankruptcy, credit_risk) with models (`.pkl`, `.pt`) and scalers.

---

### Step 2 — Backend (API)

**When:** Every time you want to use the dashboard.

**Terminal:** PowerShell, project root. **Leave this window open.**

```powershell
cd C:\Users\97254\Desktop\Finance_XAI
.\venv\Scripts\Activate.ps1
cd backend
python -m uvicorn main:app --reload --port 8000
```

**Result:** API running at http://localhost:8000 (docs: http://localhost:8000/docs)

---

### Step 3 — Frontend (dashboard)

**When:** Every time you want to use the dashboard. Backend (Step 2) must already be running.

**Terminal:** New PowerShell window (venv not required).

```powershell
cd C:\Users\97254\Desktop\Finance_XAI\frontend
npm run dev
```

**Result:** App at http://localhost:5173 — open in browser.

## Quick reference

| Step   | What        | Command (from project root) | Stays open? |
|--------|-------------|-----------------------------|-------------|
| 1      | Training    | Activate venv → `cd training` → run preprocess + `train_models.py` | No         |
| 2      | Backend     | Activate venv → `cd backend` → `python -m uvicorn main:app --reload --port 8000` | **Yes**    |
| 3      | Frontend    | `cd frontend` → `npm run dev` | **Yes**    |

**To use the app:** Start **Step 2** (backend), then **Step 3** (frontend). Open http://localhost:5173.

## What you do in the app

1. **Dashboard:** Toggle **Expert / Non-Expert**, choose **Prediction Model** (Loan / Bankruptcy / Credit Risk) and **Explainability Method** (SHAP / LIME / DiCE), enter a feature vector, click **Get prediction** and **Run X analysis**.
2. **Survey:** Open “Survey” in the nav and complete S-TIAS and SCS.

## Generating the wording (required)

Feature labels (e.g. u{201C}Your monthly incomeu{201D} vs u{201C}Applicant Incomeu{201D}) are always generated by **Gemini**. The first response per (dataset, expert/non-expert) is cached in memory for the lifetime of the backend process.

**Setup:**

1. Get an API key from [Google AI Studio](https://aistudio.google.com/apikey).
2. Set it before starting the backend:
   ```powershell
   $env:GEMINI_API_KEY = "your-api-key"
   cd backend
   python -m uvicorn main:app --reload --port 8000
   ```
3. Install the dependency if not already installed: `pip install google-generativeai`
4. Labels are generated on the first SHAP/LIME/DiCE request per (dataset, mode) and reused automatically.

If the key is not set or the Gemini call fails, the backend will return an error.

## Troubleshooting

- **Backend: "Model not found"** → Run **Step 1** (training) first.
- **Backend: "Dataset not found"** → Set `$env:XAI_DATA_DIR = "C:\Users\97254\Desktop\datasets"` and run Step 1 again.
- **Frontend can’t reach API** → Make sure Step 2 (backend) is running on port 8000.
- **"uvicorn" or "module not found"** → Activate venv and run `pip install -r backend\requirements.txt`.
